# Plan: إعادة هيكلة نظام إدارة الجداول الدراسية

## TL;DR
إعادة هيكلة كاملة للمشروع من نظام PHP إجرائي بدائي إلى نظام MVC مخصص متكامل بواجهة AdminLTE احترافية، مع قاعدة بيانات مُعاد تصميمها، ونظام صلاحيات RBAC، وسجل تدقيق، وإشعارات، وتصدير PDF/Excel، وواجهة API.

### 1.2 Front Controller + Router
- `public/index.php`: Bootstrap app, parse URL, dispatch to controller
- `core/Router.php`: Register GET/POST/PUT/DELETE routes with patterns like `/departments/{id}/edit`
- `config/routes.php`: Define all application routes in one file
- Support route groups with middleware (e.g., all `/api/*` routes get API middleware)

### 1.3 Core Classes
- `core/Database.php`: MySQLi wrapper singleton
  - `query($sql, $params)` → prepared statement execution
  - `fetch($sql, $params)` → single row
  - `fetchAll($sql, $params)` → array of rows  
  - `insert($table, $data)` → INSERT + return lastInsertId
  - `update($table, $data, $where)` → UPDATE
  - `delete($table, $where)` → DELETE
  - Transaction support: `beginTransaction()`, `commit()`, `rollback()`

- `core/Model.php`: Base model
  - Static `$table`, `$primaryKey`, `$fillable` properties
  - `find($id)`, `all()`, `where($conditions)`, `create($data)`, `update($id, $data)`, `delete($id)` 
  - `paginate($page, $perPage)` → paginated results with metadata
  - `with($relations)` → eager loading via JOINs

- `core/Controller.php`: Base controller
  - `view($template, $data)` → render view with layout
  - `redirect($url, $flash)` → redirect with optional flash
  - `json($data, $status)` → JSON response
  - `authorize($permission)` → check RBAC permission
  - `validate($rules)` → input validation

- `core/View.php`: Template engine
  - Layout inheritance: `$this->layout('layouts/app')`
  - Sections: `$this->section('content')` / `$this->endSection()`
  - Partials: `$this->include('partials/flash')`
  - Auto-escape output: `$this->e($var)`

- `core/Request.php`: HTTP request abstraction
  - `input($key, $default)`, `all()`, `has($key)`
  - `method()`, `isPost()`, `isAjax()`
  - `validate($rules)` → returns validated data or throws ValidationException

- `core/Session.php`: Session management
  - `get()`, `set()`, `has()`, `remove()`
  - `flash($key, $value)`, `getFlash($key)` → flash data
  - `regenerate()` → prevent session fixation

### 1.4 Helper Functions (`app/Helpers/functions.php`)
- `url($path)` → full URL generation
- `asset($path)` → asset URL with cache busting
- `csrf_field()` → hidden input with CSRF token
- `csrf_token()` → raw token
- `old($field, $default)` → previous form input
- `e($string)` → htmlspecialchars wrapper
- `redirect($url)` → redirect helper
- `auth()` → current authenticated user
- `can($permission)` → RBAC check shorthand
- `config($key)` → config value access

### 1.5 Middleware System
- `app/Middleware/AuthMiddleware.php` → Check session authentication
- `app/Middleware/CsrfMiddleware.php` → Validate CSRF token on POST/PUT/DELETE
- `app/Middleware/RbacMiddleware.php` → Check user permissions
- `app/Middleware/ApiAuthMiddleware.php` → API token authentication
- `app/Middleware/AuditMiddleware.php` → Log actions to audit_logs

**Depends on**: Nothing (foundation)

### 3.4 Route Definitions (config/routes.php)
Pattern: RESTful CRUD routes with middleware groups

```
// Public
GET  /login                    → AuthController@login
POST /login                    → AuthController@doLogin

// Authenticated (auth middleware)
GET  /                         → DashboardController@index
GET  /dashboard                → DashboardController@index

// CRUD Resource Pattern (each entity):
GET  /departments              → DepartmentController@index
GET  /departments/create       → DepartmentController@create
POST /departments              → DepartmentController@store
GET  /departments/{id}/edit    → DepartmentController@edit
POST /departments/{id}         → DepartmentController@update
POST /departments/{id}/delete  → DepartmentController@destroy

// ... same pattern for all resources

// API routes (api middleware)
POST /api/login                → Api\AuthController@login
GET  /api/timetable            → Api\TimetableController@index
// ... etc
```

## Decisions

- **No Composer**: Since the project runs on XAMPP/shared hosting, avoid Composer dependency. Include libraries manually (AdminLTE via CDN, PDF library via direct download)
- **MySQLi retained**: Stay with MySQLi (not PDO) for backward compatibility, but wrap in Database class
- **Gradual migration**: Keep old files during development, add redirect layer later, eventually remove
- **Single front controller**: All requests go through `public/index.php` except `public/install.php`
- **Arabic UI preserved**: Keep all Arabic text, add i18n hooks for future English support but don't implement full i18n now
- **Export without heavy deps**: CSV export built-in, PDF via DomPDF (single-file include possible) or HTML-to-PDF browser print
