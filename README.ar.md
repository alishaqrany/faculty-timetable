## نظرة عامة

هذا المشروع هو نظام لإدارة الجدول الدراسي الجامعي باستخدام PHP و MySQL. يدعم إدارة البيانات الأكاديمية الأساسية، وتوزيع المواد على أعضاء هيئة التدريس، ثم تسكين المحاضرات داخل الجدول النهائي مع منع التعارضات، بالإضافة إلى التصدير والتنبيهات والنسخ الاحتياطي.

يحتوي المستودع حاليًا على:

- تطبيق حديث نسبيًا بأسلوب MVC داخل `app/` و `core/` و `config/` و `public/`
- نسخة إجرائية أقدم داخل `_legacy/` للاحتفاظ بالتوافق والرجوع إليها عند الحاجة

نقطة الدخول الفعلية للتطبيق الحالي هي `public/index.php`.

## أهم المميزات

- تسجيل دخول وصلاحيات للصفحات المحمية
- إدارة الأقسام والفرق والشعب والمواد
- إدارة أعضاء هيئة التدريس والمستخدمين
- إدارة القاعات والفترات
- توزيع المواد على الأعضاء
- تسكين الجدول مع منع التعارضات
- نظام أولوية للجدولة
- عرض الجدول النهائي وتصديره
- سجلات تدقيق وإشعارات وإعدادات نظام
- أدوات نقل بيانات ونسخ احتياطي ومزامنة سحابية
- واجهات API لبعض بيانات النظام

## التقنيات المستخدمة

- PHP 8.0 أو أحدث
- MySQL أو MariaDB
- نواة مخصصة للتوجيه والتشغيل
- طبقة قاعدة بيانات مبنية على MySQLi
- بيئة تشغيل مثل XAMPP أو Apache أو أي استضافة PHP متوافقة

## هيكل المشروع

```text
app/            المتحكمات والنماذج والخدمات والواجهات
config/         إعدادات التطبيق وقاعدة البيانات والمسارات والصلاحيات والسحابة
core/           التهيئة والتوجيه والطلبات والجلسات وقاعدة البيانات والعرض
database/       ملفات الترحيل والبذور وأداة ترحيل البيانات القديمة
docs/           توثيق عربي وملاحظات التوافق
public/         نقطة الدخول ومعالج التثبيت
storage/        النسخ الاحتياطية والتصدير والسجلات والذاكرة المؤقتة
_legacy/        النسخة الإجرائية القديمة
```

## التشغيل المحلي

1. ضع المشروع داخل مجلد الويب المحلي، مثل `c:/xampp/htdocs/timetable`.
2. شغّل Apache و MySQL.
3. تأكد من توفر PHP 8.0 أو أحدث.
4. افتح معالج التثبيت:

   `http://localhost/timetable/public/install.php`

5. أكمل خطوات التثبيت:
   - فحص المتطلبات
   - إعداد قاعدة البيانات
   - تشغيل ملفات التهيئة والبذور
   - إنشاء حساب المدير
   - حفظ إعدادات النظام
6. سجّل الدخول من:

   `http://localhost/timetable/public/login`

## الوحدات الرئيسية

- المصادقة: تسجيل الدخول والخروج والملف الشخصي وحماية المسارات
- لوحة التحكم: الصفحة الرئيسية بعد تسجيل الدخول
- الأقسام: إدارة الأقسام الأكاديمية
- الفرق: إدارة المستويات أو الفرق الدراسية
- الشعب والأقسام الفرعية: تنظيم مجموعات الدراسة
- أعضاء هيئة التدريس: إدارة بيانات الأعضاء
- المواد: تعريف المواد بحسب القسم والمستوى
- القاعات: إدارة قاعات التدريس
- الفترات: تعريف الفترات الزمنية
- توزيع المواد: ربط المادة بعضو هيئة التدريس والشعبة
- التسكين: وضع السجلات داخل الجدول مع التحقق من التعارض
- الأولوية: التحكم في دور الجدولة والاستثناءات
- الجدول النهائي: العرض والتصدير
- المستخدمون: حسابات النظام
- سجلات التدقيق: تتبع العمليات الإدارية
- الإشعارات: إرسال ومتابعة التنبيهات
- الإعدادات ونقل البيانات: إعدادات عامة واستيراد وتصدير
- النسخ الاحتياطي والمزامنة السحابية: حفظ محلي ورفع سحابي عند التفعيل

## دورة الاستخدام

1. تثبيت النظام.
2. تسجيل الدخول بحساب المدير.
3. إنشاء الأقسام والفرق والشعب والمواد والقاعات والفترات.
4. إضافة أعضاء هيئة التدريس وحسابات المستخدمين.
5. توزيع المواد على الأعضاء.
6. إعداد نظام الأولوية إذا لزم ذلك.
7. تنفيذ التسكين داخل الجدول.
8. مراجعة الجدول النهائي أو تصديره.

## ملخص واجهات API

مسار المصادقة العام:

- `POST /api/auth/login`

المسارات المحمية:

- `GET /api/timetable`
- `GET /api/departments`
- `GET /api/members`
- `GET /api/classrooms`
- `GET /api/subjects`
- `GET /api/sections`
- `GET /api/sessions`

المسارات المحمية تحتاج Bearer Token.

## ملاحظات مهمة

- يقوم معالج التثبيت بحفظ إعدادات قاعدة البيانات داخل `config/database.php`.
- بعض الوظائف تعتمد على صلاحية كتابة داخل `storage/` مثل النسخ الاحتياطي والتصدير والسجلات وتحديد المعدل.
- إعدادات المزامنة السحابية توجد في `config/cloud.php`.
- ما زال المستودع يحتوي على ملفات توافقية أثناء الانتقال من البنية القديمة إلى الحالية.
